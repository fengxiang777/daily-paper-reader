## 摘要
键值（KV）缓存是长上下文大语言模型（LLM）推理中的主要内存瓶颈。两类领先的无训练方法在结构上均存在局限：令牌选择方法（如 SnapKV、Ada-KV）通过观察窗口评重要性并逐出低分令牌，但逐出是不可逆的——当重要性信号在查询无关重用下退化时，准确率会下降 11-15 个百分点；统一低秩编码保留每个令牌但在各处分配相同秩，浪费了预算。我们观察到，这两种失效有共同的解决之道：应当分配秩而非逐出。我们提出 VarRate，一种无训练的 KV 编解码器，它根据每个令牌的查询显著性分配可变低秩预算，使每个令牌保留非零秩。可比较的自适应秩编解码器仅通过训练才能实现这种分配；VarRate 则无需训练。由于没有令牌被丢弃，在查询感知选择发生崩溃的场景下，它的性能仅下降 3.5-5.5 个百分点。在 LongBench（16 个任务）上匹配 20% 的预算，VarRate 在 Llama-3.1-8B 和 Qwen2.5-7B 模型上都与未压缩模型相差在 0.8 个百分点以内。在这两个模型上平均而言，它是性能最强的等内存压缩器。在两个模型上，它均显著优于其统一秩消融实验。与专门为查询无关重用设计的 KVzip 相比，在四个设置中有三个达到准确率等同，总体差距在一个百分点以内，而预填充开销仅为其约八分之一。

## Abstract
The key-value (KV) cache is the main memory bottleneck in long-context large language model (LLM) inference. Two leading training-free families are both structurally limited: token-selection methods (SnapKV, Ada-KV) score importance from an observation window and evict low-scoring tokens, but eviction is irreversible -- so when the importance signal degrades under query-agnostic reuse, accuracy collapses by 11-15 points; uniform low-rank coding keeps every token but spends equal rank everywhere, wasting budget. We observe that both failures share one cure: rank should be allocated, not evicted. We present VarRate, a training-free KV codec that assigns each token a variable low-rank budget by its query salience, keeping every token at a nonzero rank. Comparable adaptive-rank codecs reach this allocation only through training; VarRate requires none. Because no token is dropped, it degrades by only 3.5-5.5 points where query-aware selection collapses. At a matched 20% budget on LongBench (16 tasks), VarRate stays within 0.8 points of the uncompressed model on both Llama-3.1-8B and Qwen2.5-7B. Averaged over the two, it is the strongest matched-memory compressor. It significantly beats its uniform-rank ablation on both models. Against KVzip, a method purpose-built for query-agnostic reuse, it is accuracy-equivalent in three of four settings and within a point overall, at about one-eighth the prefill overhead.

### 1. 论文的核心问题与整体含义
- **研究动机**：长上下文大语言模型推理中，键值（KV）缓存的内存随上下文长度线性增长，成为解码时的主要瓶颈。现存的**无训练压缩方案**主要分为两类：
  - **令牌选择与 eviction**（如 SnapKV、PyramidKV、Ada‑KV）：以不可逆丢弃方式保留高重要性令牌，但在**查询无关重用**（如缓存复用、多轮对话）时，重要性信号过时，准确率急剧崩塌（损失 11‑15 个百分点）。
  - **统一低秩编码**（如 Palu）：保留全部令牌但对所有令牌分配相同的低秩预算，导致关键令牌预算不足、平凡令牌资源浪费。
- **整体含义**：作者指出两种失败共享一个根源——**应当分配（allocate）秩，而非 evict（驱逐）令牌**。由此提出 VarRate，一种训练自由、可变速率、低秩的 KV 编解码器，旨在不丢弃任何令牌的前提下，按查询显著性动态分配秩预算，从而实现稳健且高保真的缓存压缩。

### 2. 方法论
- **核心思想**：为每个令牌分配一个**可变低秩预算**，令牌的秩与其查询显著性（通过观察窗口的注意力分数衡量）成正比，但所有令牌的秩始终**不低于一个严格正数的下限**（\(r_{\min}\)），因此没有任何令牌被彻底丢弃。
- **关键技术细节**：
  1. **预处理与联合编码**：将每个层的 key（取消 RoPE 旋转）与 value 拼接成一个全局向量 \(\mathbf{z}_t = [\tilde{\mathbf{k}}_t; \mathbf{v}_t]\)，一个共享基即可服务所有注意力头。
  2. **残差编解码**：设置一组步长锚点（anchor tokens）和最近窗口的令牌作为精确保留集合 \(E\)；其余令牌计算其与最近锚点均值的残差 \(\mathbf{e}_t = \mathbf{z}_t - \bar{\mathbf{z}}_t\)。
  3. **共享低秩基底**：离线使用少量无标注校准上下文，对各层的残差集合进行 SVD，取前 \(R\) 个右奇异向量构成基底 \(\mathbf{V} \in \mathbb{R}^{R \times D}\)。该基底**无需训练、无梯度，一次性生成并共享**。
  4. **显著性评分**：采用 SnapKV 的观察窗口注意力得分进行 min‑max 归一化，得到每个编码令牌的显著性 \(\hat{s}_t \in [0,1]\)。
  5. **水位填充分配**：在固定总预算 \(B = D(\kappa L - n_{\text{ref}})\) 下，通过迭代法求解
     \[
     r_t = \mathrm{clip}(r_{\min} + \lambda \hat{s}_t,\ r_{\min},\ R),\quad \sum_{t\in C} r_t = B,
     \]
     即高显著令牌获得高秩，其余至少保持 \(r_{\min}\)。该分配**与统一秩码本使用完全相同的总参数量**。
  6. **重建**：编码令牌通过截断投影 \(\hat{\mathbf{z}}_t = \bar{\mathbf{z}}_t + \mathbf{V}^\top (\mathbf{m}_t \odot \mathbf{c}_t)\) 重建，最后将 key 重新施加 RoPE。
- **算法流程**：预填充阶段一次完成残差计算、显著性评分、water‑filling 秩分配、投影与重建，不引入二次编码或微调。

### 6. 主要结论与发现
- **在匹配的 20% 内存下**：VarRate 在 Llama‑3.1‑8B 和 Qwen2.5‑7B 上分别与未压缩模型相差 0.3 和 0.8 分以内，两模型平均是所有等内存压缩器中**最强**。

- **预算扫描**：VarRate 在 20% 区间全面优于已发表低秩码本 Palu，并在宽松预算下优势缩小。
- **组合性**：低秩系数可进一步量化到 3‑bit（缓存占用降至 9%）而几乎无准确率损失。
- **稳健性**：基底更换、随机信号、阶跃分配等消融一致性表明，方法的核心来自**内容敏感的、渐变的秩分配**，而非单纯的形状或启发式。

### 7. 优点
- **创新性**：首次以**训练自由**方式实现逐令牌可变秩分配，填补了令牌保留与自适应压缩之间的空白。
- **根本性优势**：“分配秩而非驱逐”的设计使过时信号只会粗化令牌而不会永久丢弃，自然获得查询无关重用下的稳健性。
- **即插即用**：无需任何训练、微调或架构改动，仅需一次离线 SVD 校准，可直接应用于现有多数模型。
- **灵活组合**：与量化正交，进一步降低内存；基底共享，开销可控。
- **实验严谨**：内存匹配严格，多模型、多任务、多预算、强统计检验，并附大量消融与压力测试，支撑结论可信。
- **工程披露透明**：详细列出了实现开销分布、潜在优化路径及实验中遇到的陷阱（如 fp32 vs bf16 注意点），对社区复现友好。

### 8. 不足与局限
- **预填充开销较高**：当前未极致优化的实现带来 ~44% 的预填充时间增加，尽管算术环节极轻，但 anchor 搜索、投影等仍需 kernel 融合才能彻底降低。
- **解码阶段内存未压缩**：因重建为全尺寸缓存，需 fused kernel 才能实现实际显存节约；论文沿用该领域惯例，未提供实测解码内存节省。
- **极端预算下的脆弱性**：在 8%–10% 极低缓存保留下，VarRate 可能会落后于精确令牌选择（尤其在 GQA 头较少、基域受限时），此时低秩基底难以保留足够信息。
- **对分布式长文摘摘要弱于量化**：详尽的消融表明，低秩压缩在处理需要保存广泛、分散信息的任务（如新闻报道摘要）时，性能仍稍逊于同内存的量化方案。
- **依赖离线基底**：基底虽对校准样本不敏感，但仍需离线步骤；极端分布偏移下的鲁棒性未深入验证。
- **比较对象有限**：仅与已公布的无训练方法对比，未涉及同期训练式自适应秩方法（如 DynaKV 等），但这些方法通常需要数据与梯度，不属于同一赛道。
