## 摘要
大型语言模型（LLM）智能体通常通过更改提示、模型或手写工作流来改进，而围绕模型的执行框架被视为固定基础设施。我们认为，这个框架本身就是一个可学习的控制层。我们将框架操作形式化为有限时域框架MDP，其中轻量级控制器选择结构性执行动作，而LLM执行器保持冻结。控制器使用仅采用终端任务评估奖励的优势加权回归，从离线回放轨迹中训练。我们还将最终任务质量与事后框架成熟度分数分离开来，后者衡量框架是否遵循可靠的执行模式，而非仅仅评估最终答案是否正确。这种分离提供了框架学习的有限缓冲视角：最终质量的提升需要离线缓冲区中的高回报支持，而过程行为只要与优势加权动作对齐，就可以随时改变。在六个受控域和两个公共基准适配器上，学习到的控制器持续改进了验证行为，并选择性地提升了最终任务质量，在适配的tau-bench零售、适配的AgentBench DB-Bench以及带有校准结构化验证器的编码任务上增益最大。与行为克隆和强制检查的消融实验显示，这些增益不能由模仿或简单添加检查来解释。这些结果将框架控制确立为冻结LLM智能体的一个可学习层，同时表明离线支持限制了更好的过程控制何时转化为更好的最终答案。

## Abstract
Large language model (LLM) agents are usually improved by changing prompts, models, or hand-written workflows, while the execution harness around the model is treated as fixed infrastructure. We argue that this harness is itself a learnable control layer. We formalize harness operation as a finite-horizon Harness MDP, where a lightweight controller selects structural execution actions while the LLM executor remains frozen. The controller is trained from offline rollouts using advantage-weighted regression with only terminal task-rubric rewards. We also separate final task quality from a post-hoc Harness Maturity Score, which measures whether the harness follows reliable execution patterns rather than only whether the final answer is correct. This separation gives a finite-buffer view of harness learning: final-quality gains require high-return support in the offline buffer, while process behavior can shift whenever it aligns with advantage-weighted actions. Across six controlled domains and two public-benchmark adapters, the learned controller consistently improves verification behavior and selectively improves final task quality, with the largest gains on adapted tau-bench retail, adapted AgentBench DB-Bench, and coding with a calibrated structural verifier. Ablations against behavior cloning and Forced CHECK show that the gains are not explained by imitation or by simply adding checks. These results identify harness control as a learnable layer for frozen LLM agents, while showing that offline support limits when better process control becomes better final answers.

### 2. 方法论
#### 2.1 框架 MDP 建模
- **状态**：紧凑的轨迹特征向量，包含轨迹进度、草稿可用性、证据覆盖、工具输出、验证器反馈、近期的失败次数、剩余预算、上一动作以及领域特定的结构标志，但不包含 LLM 隐藏状态。
- **动作空间**：七个结构化控制操作：`OBSERVE`（观察）、`RETRIEVE`（检索）、`CALL-TOOL`（调用工具）、`DRAFT`（拟稿）、`CHECK`（检查）、`REVISE`（修改）、`SUBMIT`（提交）。领域适配器负责实现具体语义并屏蔽不可用动作。
- **奖励**：仅使用终端任务评分（task-rubric reward）作为回报，不提供任何中间动作偏好奖励。任务评分由领域特定的验证器根据最终产出计算，例如编码任务基于测试通过情况与约束满足度，研究任务基于答案正确性和证据支撑等。

#### 2.2 离线优势加权回归
- **训练方式**：从固定的离线回放缓冲区（包含基础与探索性框架产生的轨迹）中学习。控制器是一个单隐藏层 MLP（64 个隐藏单元，输出层为七选一的 softmax，被屏蔽的动作用极大负数遮蔽）。
- **目标函数**：加权行为克隆，权重为指数优势函数 $w=\exp(A/\beta)$，$\beta=0.2$。目标最大化 $\mathbb{E}_{(s,a)\sim\mathcal{D}}\left[w\cdot\log\pi_\theta(a|s)\right]$。
- **设计思想**：增加高回报轨迹中出现动作的对数似然，不引入新的探索，也不直接奖励特定动作模式（如 CHECK 本身）。训练使用 Adam 优化器（学习率 1e-3，批大小 256，20 个 epoch，权重裁剪至 [0.1, 10.0]，并加入熵正则系数 0.01）。

- 评估框架是否遵循可靠执行模式的事后诊断指标，从未作为训练奖励。
- 由七项过程事件加权构成：提交前检查、声明前取证、提交前测试、失败后修改、有效工具使用、充分后停止、提前提交（惩罚项）。全文实验表明，控制器主要通过提升“提交前检查”带动 HMS 改善。

#### 2.4 理论保证
- **最优策略不变性**：终端任务评分加上基于势能的中间奖励（势能整形）不会改变最优策略集合；但直接奖励特定动作（如检查）可能改变最优策略。
- **有限缓冲区解耦分析**：证明过程统计量（如验证行为）的改变由优势权重与过程统计量的协方差决定，可先于最终质量提升发生；最终质量的提高受限于缓冲区中最佳轨迹的支持程度（缓冲区松弛度 $\sigma_\mathcal{B}$）。

### 8. 不足与局限
- **离线数据依赖性**：最终质量增益严重受限于缓冲区中高回报轨迹的存在。对于某些领域，现有缓冲区缺少有效的高质量轨迹，导致学习器只能改善过程而不能提升最终分数。
- **基准适配与评分**：公共基准实验使用了适配后的评分协议，并非原生的上游官方分数，因此不能直接作为对应基准的成绩，只能作为框架控制有效性的外部验证。编码领域的增益依赖于校准后的结构验证器，而原始严格验证器下基础框架已近天花板。
- **过程诊断的局限性**：HMS 仍依靠固定阈值和简单规则（如 EarlySubmit 阈值），部分事件（如 StopWhenSufficient）始终稀疏，诊断准确性受阈值影响。
- **任务规模与可扩展性**：受控域每个仅 100 个人工标注任务，适配器仅 16 个训练任务，样本量有限。控制器状态特征和动作空间为手工设计，扩展到更复杂环境或动作空间时可能需要重新设计。
- **冷冻 LLM 限制**：LLM 执行器完全冻结，当 LLM 本身能力不足时，单纯的框架策略优化可能无法弥补模型自身的生成质量缺陷。
