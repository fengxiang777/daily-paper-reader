## 摘要
自我进化框架通常优化任务解决方案，而将周围的实验框架视为固定不变。我们提出框架感知的自我进化（HASE），一个智能体强化学习框架，其中单个模型可以在多轮动作空间中生成任务解决方案或编辑选定的框架组件。HASE使一个Qwen3-8B模型能够媲美使用Claude Code作为框架提出者的GPT-OSS-120B模型的文本分类性能。在alpha因子挖掘中，HASE超越了报道的GPT-OSS-120B基线。HASE还能修复不完善的评估组件，并在圆形装箱算法发现中收敛到最先进的性能。这些结果表明，HASE通过一个统一的智能体过程同时改进了实验框架和解决方案。

## Abstract
Self-evolving frameworks usually optimize task solutions while treating the surrounding harness as fixed. We introduce Harness-Aware Self-Evolving (HASE), an agentic reinforcement-learning framework in which a single model can generate task solutions or edit selected harness components in a multi-turn action space. HASE enables a single Qwen3-8B model to match the text-classification performance of a GPT-OSS-120B model that uses Claude Code as the harness proposer. In alpha factor mining, HASE outperforms the reported GPT-OSS-120B baseline. HASE also repairs imperfect evaluation components and converges to state-of-the-art performance in circle-packing algorithm discovery. These results show that HASE improves the harness and the solution through one unified agentic process.

*   **核心问题**：当前主流的自我进化系统（Self-Evolving Systems）通常只迭代优化 **任务解决方案**，而将**实验框架**视为固定不变的。实验框架是指模型与任务交互的所有中介组件（如提示词、记忆、检索工具、评估器）。
*   **问题严重性**：这种“固定框架”的假设存在两大缺陷：1）人为编写的评估器可能存在缺陷，导致模型在得分驱动下进行“奖励黑客”行为而非真正解决任务；2）不完善的引导组件（如提示词）可能提供劣质的搜索先验，限制模型潜力。
*   **整体含义**：论文旨在打破这一限制，提出**框架感知的自我进化 (HASE)**，其核心思想是**让模型在同一个强化学习过程中，同时进化其自身权重、任务解决方案和实验框架组件**，从而消除由固定、不完美的框架带来的性能瓶颈。

### 2. 论文提出的方法论：HASE框架

**核心思想**：HASE框架让一个单一的模型在统一的多轮动作空间中，既可以提交任务解决方案，也可以使用沙盒工具来**编辑、修改选定的实验框架组件**。

**关键技术细节**：
*   **两类可编辑的框架组件**：
    *   **引导组件 (Guidance Components)**：包括提示词、记忆格式、检索脚本等。优化它们可以改善搜索过程，且不改变任务本身的正确性。
    *   **评估组件 (Evaluation Components)**：即本地评估器。优化它需要格外小心，以防模型为获高分而削弱评估器。
*   **防止“奖励黑客”的机制——评估器修复**：
    *   HASE使用**本地评估器**提供密集的训练反馈，但只在阶段边界查询**真实世界评估器**（如排行榜）获取稀疏但准确的“神谕得分”。
    *   当本地得分与神谕得分出现分歧时，模型生成的解决方案被加入**不匹配集合 (Mismatch Set)**，作为未来评估器修复的测试用例。
    *   评估器补丁只有在修复了`M`中的不匹配并保留了必要接口时才被接受，确保评估器的进化被真实世界反馈所锚定。
*   **算法流程 (Agentic RL View)**：
    *   HASE维护一个持久化的框架`h`、模型策略`πθ`和一个阶段候选池`Cp`。

    *   奖励基于解决方案的质量。对于引导性编辑，采用**概念验证得分**进行初步筛选；对于评估性编辑，仅在发现“代理-神谕”分歧时给予奖励。
