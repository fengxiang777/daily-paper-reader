## 摘要
当推理链变长时，即便每一步都很简单，交替进行推理与工具使用的语言智能体表现也会急剧下降。我们将此归因于情境稀释：智能体的调查状态（已确认什么、怀疑什么、还需要什么）仅隐式地存在于不断增长的上下文窗口中，早期发现被后续检索所淹没。我们推出 SLEUTH，它通过结构化的认知工作记忆使该状态显式且可操作：智能体维护着与来源对应的已确认事实、按证据排序的活跃假设，以及直接驱动其下一步行动的开放问题。在五个多跳基准和五个既定基线中，SLEUTH 的优势随难度增长而扩大，从 HotpotQA 上的 +5 分到四跳链上的 +11 分，无需多次回合便超越了 Reflexion。分析剩余差距所在，我们识别出证据充分性问题：智能体常常找到答案但未能决断，将预算浪费在多余的验证上。一个轻量级的决断触发器解决了这个问题，但仅当智能体已维护结构化状态时才有效：将相同触发器应用于非结构化智能体则无任何改善，这表明有组织的认知状态是有效决断的必要条件。最后，在较弱模型上强制协议遵循，可在最难问题上回收高达 +19 分的提升，表明智能体组织推理的方式，而非原始模型能力，是扩展多跳推理的关键要素。

## Abstract
Language agents that interleave reasoning and tool use degrade sharply as reasoning chains lengthen, even when each individual step is easy. We trace this to context dilution: an agent's investigative state (what it has confirmed, what it suspects, and what it still needs) lives only implicitly in a growing context window, where early discoveries are buried under later retrievals. We introduce SLEUTH, which makes this state explicit and actionable through a structured epistemic working memory: the agent maintains Confirmed Facts grounded to sources, Active Hypotheses ranked by evidence, and Open Questions that directly drive its next action. Across five multi-hop benchmarks and five established baselines, SLEUTH's advantage grows with difficulty, from +5 points on HotpotQA to +11 on 4-hop chains, surpassing Reflexion without multiple episodes. Analyzing where the remaining gap lies, we identify the evidence sufficiency problem: agents often find the answer but fail to commit, exhausting their budget on needless verification. A lightweight commitment trigger fixes this, but only when the agent already maintains structured state: the identical trigger applied to an unstructured agent yields no improvement, isolating organized epistemic state as the necessary condition for effective commitment. Finally, enforcing protocol adherence on a weaker model recovers up to +19 points on the hardest problems, showing that how an agent organizes its reasoning, not raw model capability, is the active ingredient for scaling multi-hop reasoning.

### 方法论：SLEUTH 结构化认知工作记忆
- **核心思想**：SLEUTH（Structured Layered Evidence-Updated Tracking of Hypotheses）通过纯提示工程，为语言智能体引入显式的认知工作记忆，将调查状态分解为三类结构化组件，并在每一步更新和输出。
- **工作记忆结构**（每回合维护）：
  - **已确认事实 (Ft)**：从工具输出中提取的、带有来源证据的已验证陈述，单调累积，不可撤销。
  - **活跃假设 (Ht)**：候选答案，按证据支持度排序，链向支持/矛盾的事实，并标注置信度（高/中/低）。
  - **开放问题 (Qt)**：当前最需解决的不确定性，每个问题附有提议动作与优先级，优先级最高的开放问题直接决定下一工具调用。
- **更新流程**：每次工具调用后，依次完成：提取新事实 → 更新假设（调整置信度、生成新假设）→ 更新问题（移除已回答的，生成新问题以区分假设）→ 选择动作（执行最高优先级问题的动作）。当 Qt 为空且有假设为高置信度时，智能体提交答案。
- **承诺机制（Nudge）**：针对“证据充分性”问题（智能体已找到答案却过度验证、耗尽预算），引入轻量级触发器：当回合数达到总预算的 70%（α=0.7）且已确认事实数不少于 2 时，强制智能体利用已累积的结构化状态进行答案综合，不再继续探索。该机制仅在配合结构化工作记忆时有效。
- **实现形式**：系统提示构成，无训练、无架构修改；工具接口所有方法统一（search、lookup、finish）；可通过提示适配与运行时格式校验提升低指令遵循度模型的协议遵从率。

### 实验设计

  - MuSiQue（2跳/3跳/4跳可回答子集，各 1252/760/405 题，干扰项 20 段）

  - 均使用全段落上下文提供检索，评估标准为标准化后的精确匹配（EM）。

### 主要结论与发现
- SLEUTH 在所有基准上均取得最高 EM，且优势随推理难度扩大：在 HotpotQA 上 +4.7，在 MuSiQue 4跳上 +10.9，超过需多次回合的 Reflexion。
- 结构化事实追踪是防止信息丢失的主导力量，已可回收大部分提升；开放问题与假设需协同工作才有额外增益。
- 证据充分性问题被明确识别：智能体在找到答案后仍未决断，产生过量验证。承诺机制可转换其优越的推理能力为任务表现，但该机制仅在状态已结构化时才生效。
- 协议遵循性是关键活性成分：仅靠强制执行工作记忆结构，弱模型 GLM-5 在 4跳任务上可比未组织的强模型 Sonnet 高出近 9 个点，并在最难题上回收最高 +19 点。
- 核心启示：如何组织推理比模型能力本身更重要；认知状态管理是扩展多跳推理的关键设计维度。

### 优点
- **轻量且通用**：完全基于提示，无需训练、搜索或额外基础设施，可适配任意指令遵循模型。
- **显式可解释**：每一回合输出完整认知状态，推理过程透明，便于人类监督与调试。
- **严谨的消融与因果分析**：通过组合消融、Nudge 隔离、协议强制执行等实验，清晰地分离出结构化状态管理与承诺机制各自的因果效应。
- **问题定义深刻**：将性能下降精确定位为上下文稀释与证据充分性麻痹两种可解决的故障模式，并给出针对性的机制设计。

### 不足与局限
- **令牌开销**：结构化工作记忆增加输出长度，简单问题上相对成本较高（HotpotQA 约 1.5 倍 API 成本），虽在困难问题上趋于收敛。
- **阈值敏感性**：承诺时机（预算占比 70%）基于当前基准调参，对于探索/利用比例极端的任务可能需要重新调整。
- **单调事实累积风险**：已确认事实不可撤销，早期错误确认（如语境上正确但推理误导的陈述）可能被固化，影响后续推导。在存在矛盾信息的开放域设定中风险更大。
- **封闭检索环境**：实验在提供段落的封闭上下文内进行，未涉及开放域中时间敏感或矛盾来源的处理，局限了真实世界泛化结论。
- **错误瓶颈**：剩余错误主要为“在正确获取的证据上推理错误”，SLEUTH 仅解决信息组织问题，不提升基础推理质量，需结合验证增强等方法。
