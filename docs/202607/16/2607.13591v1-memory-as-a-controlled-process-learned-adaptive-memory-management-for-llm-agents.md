## 摘要
大语言模型（LLM）智能体越来越依赖外部记忆系统来积累跨任务的经验。然而，几乎所有现有方法，从图结构记忆到反思洞察存储，都通过固定的、手工设计的启发式规则来访问记忆。我们认为，这种静态的记忆观是智能体学习的核心瓶颈，因为最优的记忆行为本质上是上下文依赖的。任务的早期阶段受益于最少的检索，因为记忆稀疏；重复的目标类型受益于计划复用，而非通用的最近邻查找；陷入困境的智能体受益于使用替代查询进行重新检索；在长任务流中，记忆存储本身必须被整合和修剪以保持有用。我们提出了记忆作为受控过程（MemCon），这是一个将记忆操作建模为马尔可夫决策过程的框架，并学习一种在线策略，该策略自适应地决定何时检索、检索什么、检索多少，何时注入精炼的计划，以及何时整合或遗忘。MemCon是后端无关的：它封装任何现有的记忆实现，通过逐任务二元反馈学习，无需预训练和额外的LLM调用，并使用轻量级的表格型上下文赌博机与UCB探索，在几十个任务内就能收敛。在6个基准测试、3个智能体框架和3个LLM主干上，MemCon在任务成功率上持续超过多个记忆基线，最高提升15.2个百分点，同时将token消耗降低5%到20%。

## Abstract
Large Language Model (LLM) agents increasingly rely on external memory systems to accumulate experience across tasks. Yet nearly all existing approaches, from graph-structured memories to reflective insight stores, access memory through fixed, hand-designed heuristics. We argue that this static view of memory is a core bottleneck for agentic learning because optimal memory behavior is fundamentally context-dependent. The early stages of the tasks, benefit from minimal retrieval because memory is sparse; recurring goal types benefit from plan reuse rather than generic nearest-neighbor lookup; stuck agents benefit from re-retrieval with alternative queries; and across long task streams, the memory store itself must be consolidated and pruned to remain useful. We present Memory as a Controlled Process (MemCon), a framework that models memory operations as a Markov Decision Process and learns an online policy that adaptively decides when, what, and how much to retrieve, when to inject a distilled plan, and when to consolidate or forget. MemCon is backend-agnostic: it wraps any existing memory implementation, learns from task-by-task binary feedback with no pretraining and no additional LLM calls, and uses a lightweight tabular contextual bandit with UCB exploration that converges within tens of tasks. Across 6 benchmarks, 3 agent frameworks, and 3 LLM backbones, MemCon consistently outperforms multiple memory baselines by up to 15.2 points in task success while reducing token consumption by 5--20%.

### **1. 论文的核心问题与整体含义（研究动机和背景）**

*   **核心问题**：现有的大语言模型（LLM）智能体所依赖的外部记忆系统，其访问和管理方式通常是**静态且基于固定启发式规则**的（例如，固定的检索数量 top-k、固定的图搜索深度等）。这种“一刀切”的方法无法适应任务的不同阶段和上下文变化，构成了智能体持续学习和性能提升的**核心瓶颈**。
*   **研究动机**：最优的记忆行为是**上下文依赖的**。具体体现在：
    *   **任务早期**：记忆库稀疏，应减少无意义的检索。
    *   **重复性任务**：应优先复用过去的成功计划，而非进行泛化的最近邻检索。
    *   **智能体卡顿**：当重复执行无效动作时，应触发带有替代查询的再检索。
    *   **长任务流**：记忆须定期整合与遗忘，以保持有效性和避免信息过载。
*   **整体含义**：该工作主张将记忆视为一种**可学习的受控过程**，而非一个静态的工具调用管道。通过学习一个自适应记忆控制器，可以动态、最优地决定对记忆的各类操作，从而突破现有瓶颈，让智能体在各类任务中表现得更好，同时消耗更少的计算资源。

### **2. 论文提出的方法论：核心思想、关键技术细节、算法流程**

#### **核心思想**
MemCon（Memory as a Controlled Process）提出了一个**后端无关的记忆控制框架**，它不是在发明一个新记忆存储库，而是在任何现有记忆后端之上增加一个**轻量级的自适应控制层**（wrapper）。这个控制层将记忆操作（检索、注入计划、再检索、整合、遗忘、无操作）的选择过程建模为一个**马尔可夫决策过程（MDP）**，并实时学习最优的策略。

#### **关键技术细节**
*   **记忆管理MDP建模**：
    *   **状态**：融合任务进度（目标类型、步骤阶段、是否卡顿等）和记忆状态（大小、是否有可用计划、学习阶段等），并离散化为少量哈希键值。
    *   **动作**：一个预定义的有限动作集合，每个动作指定了操作类型（如 RETRIEVE、PLAN_INJECT、FORGET）及其参数（如 top-k 检索数量）。
    *   **奖励**：基于任务是否成功以及完成效率的二元反馈。
*   **在线策略学习**：
    *   **算法**：使用**表格型上下文赌博机**算法，结合**上置信界（UCB）探索**策略。
    *   **特点**：
        *   **零预训练和零额外LLM调用**：策略的决策是毫秒级的查表操作，不涉及额外的LLM推理。
        *   **暖启动先验**：Q表使用可解释的先验知识进行初始化（如检索通常有益，遗忘通常有风险），加速收敛。
        *   **信用分配**：采用**反向折扣蒙特卡洛回报**更新Q值，使得越接近任务完成最后一步的决策获得越高的奖励或惩罚。
*   **后端无关封装器与增强操作**：
    *   封装器拦截记忆的“检索”和“存储”入口，根据策略选择调用底层后端。
    *   **泛化计划注入（PLAN_INJECT）**：将特定任务成功轨迹中的实例替换为通用占位符（如“柜子3”→“[柜子]”），形成可复用的计划模板，并以JSON索引存储。
    *   **目标分解（启发式）**：对于“将两个手机放在桌上”这类复合任务，自动注入“先完成物体1的所有步骤，再对物体2重复”的引导性文本。

### **7. 优点：方法或实验设计上的亮点**

*   **思想创新**：首次将LLM智能体的记忆管理明确为一个可在线学习的顺序决策问题（Memory MDP），将“如何访问记忆”纳入学习范畴。
*   **架构优雅**：提出的后端无关封装器实现了**控制逻辑与存储逻辑的完全解耦**，任何现有或未来的记忆系统均可“即插即用”，并获得自适应控制能力。
*   **成本极低**：采用轻量级的表格型UCB学习，完全避免了为记忆管理所增加的额外LLM推理成本，非常适用于对延迟和费用敏感的实际部署场景。

*   **“多赢”结果**：同时实现了更高的任务成功率和更低的Token消耗，证明了自适应控制的根本优势在于**在正确的时间做正确的事**，而非简单地堆砌计算。

### **8. 不足与局限：包括实验覆盖、偏差风险、应用限制等**

*   **状态空间离散化的泛化性**：依赖于手工设计的离散状态特征，将其推广到全新、未见过的复杂环境时，可能需要对状态特征和离散化阈值进行重新设计。
*   **学习效率的天花板**：表格型方法决定了Q表大小会随离散状态-动作对的数量线性增长，其收敛需要每个状态被访问到，对于状态空间巨大的任务可能无法学习。
*   **“卡顿”指标的局限性**：将连续重复相同物理动作作为判断卡顿的唯一标准，可能遗漏其他形式的任务停滞（如陷入无效循环但每次输出略有不同）。
*   **对超参数的依赖**：虽然进行了敏感性分析，表格型学习方法对UCB系数、学习率等仍有一定的敏感性，不同任务类型可能需要调优。
*   **未讨论的安全与对齐风险**：论文未涉及自适应记忆操作（如遗忘）可能带来的安全风险，例如恶意攻击者可能诱导策略学习遗忘掉关键的安全准则或行为规范。
