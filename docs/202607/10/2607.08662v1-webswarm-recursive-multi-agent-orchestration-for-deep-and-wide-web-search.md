## 摘要
基于大语言模型（LLM）的网页搜索智能体正在将信息寻求从简单的问答转变为复杂的、深度与广度并重的搜索和研究型任务。单个ReAct风格的智能体受限于单条长轨迹和有限的上下文，难以同时处理深度和覆盖范围。现有多智能体系统通过并行执行和聚合来提高搜索覆盖范围，但在递归深度、协作适应性和基于证据的扩展方面仍存在明显局限。我们提出了WebSwarm，一种渐进式递归委托框架，在推理过程中联合构建任务分解、递归扩展和智能体协作。WebSwarm动态实例化智能搜索节点，每个节点将局部目标与搜索模式耦合，该模式指定节点应如何组织搜索和协作。每个节点可以自行解决其目标，或进一步委托子节点；解决后，它将证据和结果向上返回，使得父节点能够进一步扩展、修正或聚合搜索过程。为指导这一过程，WebSwarm首先探查任务相关信息在网页上的组织方式，以支撑后续节点扩展，并在同质兄弟节点间重用过程级经验。在BrowseComp-Plus、WideSearch、DeepWideSearch和GISA上的实验表明，WebSwarm在深度、广度和交织的深度与广度任务上均一致优于单智能体和多智能体基线方法。进一步的消融分析、任务难度、网页工具效率和模型泛化性分析解释了WebSwarm的有效性，并为多智能体搜索系统提供了见解。

## Abstract
Large language model (LLM)-based web search agents are transforming information seeking from simple factoid question answering into complex, deep-and-wide search and research-oriented tasks. A single ReAct-style agent is constrained by one long trajectory and limited context, making it difficult to handle depth and coverage simultaneously. Existing multi-agent systems improve search coverage through parallel execution and aggregation, but still exhibit clear limitations in recursive depth, collaboration adaptability, and evidence-grounded expansion. We propose WebSwarm, a progressive recursive delegation framework that jointly constructs task decomposition, recursive expansion, and agent collaboration during inference. WebSwarm dynamically instantiates agentic search nodes, each coupling a local objective with a search mode that specifies how the node should organize search and collaboration. Each node can either solve its objective itself or further delegate child nodes; after solving, it returns evidence and results upward, enabling parent nodes to further expand, revise, or aggregate the search process. To guide this process, WebSwarm first probes how task-relevant information is organized on the web to ground subsequent node expansion, and reuses process-level experience across homogeneous sibling nodes. Experiments on BrowseComp-Plus, WideSearch, DeepWideSearch, and GISA show that WebSwarm consistently outperforms single-agent and multi-agent baselines on deep, wide, and interleaved deep-and-wide tasks. Further analyses of ablation, task difficulty, web tool efficiency, and model generalization explain WebSwarm's effectiveness and provide insights for multi-agent search systems.

### 2. 论文提出的方法论

*   **核心思想**：
    `WebSwarm`的核心是一个**渐进式递归委托框架**。它不是预先设定一个固定的任务分解图或多代理协作拓扑，而是在推理过程中动态实例化“智能搜索节点”，并由节点自行决定是直接解决问题，还是递归地委托给子节点，并组织子节点间的协作结构。

*   **关键技术细节**：
    1.  **搜索节点 (Search Node) 与搜索模式 (Search Mode)**：
        *   每个节点是一个代理，接收一个**局部目标**和一个**搜索模式**。
        *   搜索模式定义了该节点解决问题的协议，WebSwarm定义了四种模式以应对不同瓶颈：
            *   `atom`: 原子事实查找，适合聚焦的单实体查询，节点直接调用Web工具。
            *   `deep`: 深度搜索，适合目标未知的约束谜题。采用**串行的“搜索-验证”** 结构，迭代派遣搜索者提出候选，并由验证者独立核验。
            *   `wide`: 广度搜索，适合对大量对象收集同类信息。采用**并行的“分而治之”** 结构，并发调度独立子节点并聚合结果。
            *   `entity_collect`: 实体收集，适合边界未知的枚举任务。并发调度多个`atom`节点从不同维度召回，然后合并、去重、验证。

    2.  **递归委托与证据反馈**：
        *   父节点根据自身目标和中间证据（子节点返回的结果）决定是否生成新的子节点委托、修正搜索方向或向上聚合结果。
        *   整个系统形成一棵动态生长的委托树，自顶向下委托，自底向上反馈，实现推理时的搜索结构演化。

    3.  **两种引导信号**：
        *   **外部网页结构探查 (Web-Structure-Guided Expansion)**：
            *   在需要广度扩展的节点，先派遣一个`Web-Probing Agent`。
            *   此探针代理进行轻量级搜索和页面浏览，判断任务相关信息在Web上是**集中**在少数聚合页，还是**分散**在按特定维度（如时间、实体）组织的页面中。
            *   父节点根据探查到的结构提示来规定后续节点的扩展方向、粒度和范围，避免与网络信息结构错位。
        *   **内部兄弟节点经验复用 (Experience-Guided Node Solving)**：
            *   对于`wide`节点下的一批同质子节点，先执行一小部分“侦察”节点，从其完整的搜索轨迹中提取过程级经验（如有效查询模式、可靠信息源、应避免的路径）。
            *   将提取的经验注入到剩余的子节点求解过程中，以提升求解质量和一致性。

*   **算法流程（文字说明）**：
    1.  **根节点**接收原始任务`q0`。
    2.  节点`v`根据其局部目标`qv`、搜索模式`mv`和当前收集的子结果`Rv`，判断是否需要扩展。
    3.  若需要扩展，生成子委托集`Cv = {(qi, mi)}`，即实例化一组带有目标和模式的子节点。
    4.  **并行处理子节点**：子节点`vi`根据其模式`mi`独立解决目标。对于`wide`节点下的同质子节点，可先执行“侦察”节点并提取经验，用于指导后续节点。
    5.  子节点将结果`ri`返回给父节点，父节点更新证据状态`Rv ← Rv ∪ {ri}`。
    6.  父节点基于更新的证据，决定继续委托、修正或聚合结果`rv = Aggregate(qv, Rv)`，并将结果向上返回。
    7.  重复步骤2-6，直到根节点获得最终答案。

### 7. 优点

*   **创新性协作范式**：将多代理协作从“固定拓扑下的任务分配”提升为“基于证据的递归委托”，使系统能像树一样生长，能灵活应对深度和广度的复杂交织。
*   **目标与策略耦合的设计**：通过为节点定义局部目标和搜索模式，将任务分解与求解策略紧密结合，解决了不同子任务瓶颈需要不同协作方式的难题。
*   **紧贴信息源头的启发式设计**：独创的“网页结构探查”机制，使任务的分解从依赖语义转向依赖网络信息的真实组织形式，这是一个极具工程价值的洞察，有效避免了无效检索。
*   **全面的实验评估**：实验不仅覆盖了多个专门的深度/广度基准，还包含了多维度的消融分析、层级性能分析和案例研究，论证过程严谨扎实。

### 8. 不足与局限

*   **局限一：依赖预训练LLM能力**：框架依赖于基座LLM的理解和推理能力来进行节点委托、模式选择和经验提取，未涉及针对这些步骤的专门训练或策略优化，可能未能完全挖掘模型潜能。
*   **局限二：资源消耗较高**：尽管能自适应调整，但相较于单一`ReAct`代’理，`WebSwarm`系统的委托和并行机制不可避免地会带来更多的`LLM`调用和网页工具请求，导致更高的推理成本和延迟。
*   **局限三：模态覆盖不足**：当前研究仅聚焦于基于文本的Web工具，尚未扩展到多模态（图片、视频、音频）的网络信息寻求场景。
