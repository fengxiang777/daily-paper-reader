## 摘要
大型语言模型和多模态大型语言模型的快速发展加速了能够操作日常工具并在真实世界环境中协助用户的主动式智能体的出现。然而，现有基准测试难以有效评估此类智能体，因为它们通常依赖于沙盒环境和单轮评估范式。此外，它们基于场景的任务分类将多种模型能力混合在同一任务类别中，使得难以识别智能体失败的根本原因。为了解决这些局限性，我们引入了UniClawBench，这是首个以能力为驱动、旨在评估动态真实世界环境中主动式智能体的基准测试。UniClawBench围绕五项基础模型能力构建：技能使用、探索能力、长上下文推理、多模态理解和跨平台协调。基于这些能力，我们设计了400个双语真实世界任务。不同于以往依赖静态预记录答案的基准测试，我们的基准在实时Docker容器中使用细粒度的逐步完成检查点来评估智能体。此外，我们设计了一种闭环评估策略，包括一个执行器智能体、一个隐藏的监督智能体和一个用户智能体，以模拟真实的多轮人类反馈，同时不泄露评分标准。为了将基础模型能力与框架级设计选择分离开，我们在多个智能体框架下评估了最先进的模型。通过跨模型和框架的全面比较，我们展示了基础模型能力和智能体框架设计如何共同塑造真实世界环境中的表现。为了促进未来的研究，我们公开了基准和代码，网址为https://github.com/HKU-MMLab/UniClawBench。

## Abstract
The rapid development of large language models and multimodal large language models has accelerated the emergence of proactive agents capable of operating everyday tools and assisting users in real-world environments. However, existing benchmarks struggle to evaluate such agents effectively, as they often rely on sandboxed environments and single-turn evaluation paradigms. Moreover, their scenario-based task taxonomies mix multiple model capabilities within the same task category, making it difficult to identify the root causes of agent failures. To address these limitations, we introduce UniClawBench, the first capability-driven benchmark designed to evaluate proactive agents in dynamic, real-world settings. UniClawBench is built around five foundational model capabilities: Skill Usage, Exploration, Long-Context Reasoning, Multimodal Understanding, and Cross-Platform Coordination. Based on these capabilities, we design 400 bilingual real-world tasks. Unlike previous benchmarks that rely on static, pre-recorded answers, our benchmark evaluates agents in live Docker containers using fine-grained, step-by-step completion checkpoints. Furthermore, we design a closed-loop evaluation strategy comprising an executor agent, a hidden supervisor agent, and a user agent to simulate realistic multi-turn human feedback without leaking grading criteria. To disentangle base model capabilities from framework-level design choices, we evaluate state-of-the-art models under multiple agent frameworks. Through comprehensive comparisons across both models and frameworks, we show how base model capabilities and agent framework designs jointly shape performance in real-world environments. To facilitate future research, we make our benchmark and code publicly available at https://github.com/HKU-MMLab/UniClawBench.

# UniClawBench 论文深度分析与总结

## 1. 研究动机与核心问题
*   **背景与问题**：随着大语言模型（LLM）和多模态大模型（MLLM）的发展，能够操控浏览器、终端等日常工具的“主动式智能体”迅速涌现。然而，现有评估基准存在三大结构性缺陷，难以有效衡量这类智能体在真实世界中的能力：
    *   **环境过于简化**：依赖自托管镜像或沙盒环境，无法反映现实的动态与复杂性。
    *   **评估范式单一**：采用单轮评估，忽略了真实人机交互中多轮迭代和用户反馈的闭合回路。
    *   **任务分类模糊**：按应用场景（如“办公”、“科研”）组织任务，导致多种能力混杂，当模型失败时无法确定根本原因（是视觉感知弱、长上下文推理差还是工具使用不当）。
*   **研究目标**：构建首个**能力驱动**的主动式智能体基准测试，在动态、真实世界的环境中系统地解耦和诊断智能体的关键能力缺陷。

## 2. 方法论：核心思想与关键技术
为构建能适应真实环境动态变化、模拟闭合反馈回路而不泄露评估标准的基准，UniClawBench 提出了以下核心设计：

*   **能力驱动的任务分类法**：将任务按五个可诊断失败根源的基础能力维度划分，每个任务主要瓶颈于一个维度：
    1.  **技能使用（Skill Usage）**：选择、检查并操作任务特定的工具或 API。
    2.  **探索能力（Exploration）**：在不确定、嘈杂或信息不完整的环境中进行开放式调查。
    3.  **长上下文推理（Long-Context Reasoning）**：从异构、分散的多源信息中整合、追踪证据并保持全局一致性。
    4.  **多模态理解（Multimodal Understanding）**：从图像、视频、音频中提取、解释并生成基于非文本信息的内容。
    5.  **跨平台协调（Cross-Platform Coordination）**：在浏览器、GUI 应用、本地文件等多个异构应用界面间同步状态、传递信息并协调行动。
*   **动态环境下的细粒度评估机制**：放弃传统静态的预记录答案。在实时运行的 Docker 容器中，为每项任务设计了**细粒度的、分步骤完成的检查点**，以应对真实网页或 API 内容变动的挑战。
*   **三角色闭环评估策略**：通过“信息防火墙”分离角色以模拟真实多轮反馈且不泄露答案或评分标准：
    *   **执行器智能体（Executor Agent）**：被评估对象，仅获知公开任务指令和资源，在实时环境中执行并产生中间轨迹和最终产物。
    *   **隐藏的监督智能体（Supervisor Agent）**：访问隐藏的评测标准、地面真值参考以及执行器的可见轨迹和产物，进行结构化评判（如通过/失败/继续），并输出一个有限的状态信号，不暴露具体细节。
    *   **用户模拟器（User Simulator）**：仅接收公开的可见轨迹和来自监督者的粗略进度信号，模拟真实终端用户的口吻生成自然语言反馈，驱动执行器进行下一轮迭代。

## 3. 实验设计
*   **基准与数据集**：
    *   **任务构成**：包含 400 个**双语（中英文）**手工构造的真实世界任务，每个能力维度各 40 个英文和 40 个中文任务。
    *   **任务形式**：以完整的“任务包”形式呈现，包含用户指令、初始资源、工具/技能声明、预期产出及隐藏的评测参考。任务覆盖办公、消费、创意、数据科学、软件工程等十余种应用场景，输入输出格式极为多样（JSON、视频、日历、GUI 状态等）。
    *   **运行环境**：全部任务在隔离的 Docker 容器中实时执行，内置真实浏览器、命令行、GUI 应用和本地文件系统。
*   **对比方法与配置**：进行两组系统的实验以分离模型能力与框架影响：
    1.  **跨模型对比**：在 **OpenClaw 框架**下统一评估 10 款前沿模型，包括 `Claude Opus-4.8`、`GPT-5.4`、`Kimi-2.6`、`Qwen-3.5-Plus`、`Gemini-3.1-Pro` 等。
    2.  **跨框架对比**：选取 3 款代表性模型（`GPT-5.4`、`Claude Opus-4.8`、`Kimi-2.6`），分别在 **OpenClaw、EDICT、Nanobot** 三个主流主动式智能体框架下进行评估。
*   **评估指标**：
    *   **通过率（Pass Rate, PR）**：任务最终达到“通过”状态的百分比。
    *   **平均分（Average Score, AS）**：监督者根据检查点评分的平均得分。
    *   报告方法可靠性的**人工验证**：随机抽取 50 条轨迹由三位人类专家独立评估，并与自动评分结果进行一致性比较（通过率一致率为 92.0%，AS 得分相关性强）。

## 4. 资源与算力
*   论文**未明确提及**大模型训练所需的 GPU 型号、数量或训练时长。实验部分仅说明了用于承载 Docker 容器的宿主机的硬件配置：**Intel Core i7-13700 主机，16GB 内存，每个容器分配 2GB 内存**。评测的重点在于智能体执行任务和推理时的性能，而非模型的训练成本。每个任务的平均运行时间约为 **17.4 分钟**。

*   **实验规模充足，覆盖全面**：

    *   分析维度多维，不仅展示了总体 PR/AS，还深入剖析了五个能力维度、三种框架以及多轮交互中的性能演进，并提供了 Token 消耗分析。
    *   通过随机抽样人工评估验证了自动评估管道的可靠性，确保了实验结论的客观性与可复现性。
    *   实验设计公平：通过“信息防火墙”防止评估标准泄露给被测系统；通过为所有框架安装相同的基础技能集合（如浏览器技能、搜索技能）来控制变量，确保框架间的比较相对公平。

## 6. 主要结论与发现
1.  **整体任务难度极高**：即使是表现最好的 `Claude Opus-4.8`，其总体通过率也不到 50%（47.5%），揭示了沙盒性能与真实世界操作之间的巨大鸿沟。
2.  **框架设计影响力显著**：**框架架构对性能的影响往往大于基础模型选择**。例如，集中式单智能体架构 `OpenClaw` 因上下文保持完整而通过率最高；多智能体架构 `EDICT` 尽管平均分高，但因“协调摩擦”而通过率较低；轻量级架构 `Nanobot` Token 消耗极低，但难以支撑长证据链的严格任务。
3.  **“半途而废”现象普遍**：多数模型的平均分（AS）与最终通过率（PR）之间存在巨大差距，表明智能体在长链条执行中经常犯不可恢复的错误，缺乏可靠性。
4.  **核心能力瓶颈**：**长上下文、多模态和跨平台**任务是最大挑战，而技能使用和探索能力的相对表现较好。这表明当前智能体擅长局部工具操作，但在长距离记忆、多模态证据支持和跨应用协调方面存在显著短板。
5.  **闭环交互有效**：多轮用户反馈能持续提升智能体表现，验证了闭环评估对实现动态错误恢复和模拟真实交互的必要性。

## 7. 优点与亮点
