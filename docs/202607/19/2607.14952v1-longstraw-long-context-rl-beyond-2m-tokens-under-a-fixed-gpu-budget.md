## 摘要
推理上下文长度与强化学习后训练之间日益扩大的差距：推理系统正接近百万标记的上下文长度，而后训练工作负载通常停留在256K标记或以下，并在部署时依赖长度泛化。这一差距对于AI智能体尤为重要，其观察、工具输出、文档和先前的决策会在长时间轨迹中累积。LongStraw是一个架构感知的执行栈，用于在固定GPU预算下进行百万标记的强化学习后训练，并以分组相对策略优化（GRPO）实例化。它在前向传播中评估共享提示时不启用自动求导，仅保留后续标记所需的模型特定状态，并一次重放一个短响应分支，从而以额外的重放时间为代价减小实时训练图的大小。我们将其实现用于混合循环和全注意力的Qwen3.6-27B以及压缩注意力混合专家模型GLM-5.2。在八块H20 GPU上，LongStraw针对2和8的分组，在2.1M个位置完成了Qwen的分组评分和响应后向传播；增加分组大小仅增加0.21 GB的峰值分配内存，而单独的极限测试达到了4.46M个位置。在32块H20 GPU上，我们验证了LongStraw在2.1M标记提示下，跨GLM-5.2的78层全栈端到端执行路径。这些实验建立了执行容量，而非完整的训练正确性，因为捕获的提示状态被分离，并且一些分布式前向和梯度组合路径仍未完成。

## Abstract
A growing gap separates inference context lengths from RL post-training: inference systems are approaching million-token contexts, while post-training workloads often remain at 256K tokens or below and rely on length generalization at deployment. The gap is especially important for AI agents, whose observations, tool outputs, documents, and prior decisions accumulate over long trajectories. LongStraw is an architecture-aware execution stack for million-token RL post-training under a fixed GPU budget, instantiated with Group Relative Policy Optimization (GRPO). It evaluates the shared prompt without autograd, retains only model-specific state needed by later tokens, and replays short response branches one at a time, reducing the live training graph at the cost of additional replay time. We implement it for the hybrid recurrent and full-attention Qwen3.6-27B and the compressed-attention mixture-of-experts GLM-5.2. On eight H20 GPUs, LongStraw completes grouped Qwen scoring and response backward at 2.1M positions for groups of 2 and 8; increasing the group size adds only 0.21 GB of peak allocated memory, while a separate stress test reaches 4.46M positions. On 32 H20 GPUs, we validate the end-to-end LongStraw execution path for a 2.1M-token prompt across all 78 layers of GLM-5.2. These experiments establish execution capacity rather than complete training correctness because the captured prompt state is detached and some distributed forward and gradient composition paths remain incomplete.

**研究动机**：当前，大语言模型的推理系统已支持百万标记（token）级的长上下文，但强化学习（RL）后训练的上下文长度仍停留在256K以下，依赖部署时的“长度泛化”来应对长序列。这一断层对需要累积长期观察、工具输出与决策历史的AI代理（agent）尤为致命，因为其训练无法利用完整轨迹。核心瓶颈在于：训练时需对同一长提示下的多个响应进行评分和反向传播，导致包含整个上下文的前向计算图与后向状态常驻GPU，显存爆炸。

**论文含义**：文章不追求“最长上下文”或“最快训练”，而是针对**固定GPU预算（例如8块H20）**，提出一个名为LongStraw的架构感知执行栈，通过重新设计状态生命周期和物理所有权，将GRPO（组相对策略优化）的训练上下文从无法完成的超长序列压缩到可执行的范畴，从而降低长上下文训练的门槛。本质上，它解决的是“在有限资源下，能跑多大的GRPO训练图”这一系统可行性问题。

## 2. 方法论

### 核心思想
将GRPO的一次更新拆分为**一次共享提示的无梯度捕获**与**多个短响应分支的串行重放**：
- **捕获阶段**：对共享的长提示进行一次前向传播，但**关闭自动求导**，仅保留后续响应所需的**架构特定状态**（例如注意力键值页、循环状态、MLA潜在向量），而丢弃所有中间激活、专家路由缓冲等。
- **重放阶段**：按顺序重建每一个短响应的计算图，利用已存储的提示状态计算条件概率，进行反向传播并累积梯度，然后立即释放当前响应的计算图。
- **最终优化**：全部响应完成后，对累积的梯度调用一次优化器更新参数。此时存储的提示状态已过时。

### 关键技术细节与算法流程
**图边界变更**：传统方法中，实时自动求导图跨域 `[提示 + 响应]` 序列；LongStraw将边界收缩到单个 `[响应]` 序列。公式 `∇θL = ∂L/∂θ|zP + (∂L/∂zP)·(∂zP/∂θ)` 表明，文中方法仅计算第一项，**提示状态的梯度被刻意分离（detached）**，第二项未计算。

**架构感知的状态库存与所有权**：
- **Qwen（密集混合注意力）**：64层解码器，48个循环层（Gated DeltaNet）和16个全注意力层。捕获后保留压缩的循环状态和紧凑的物理KV页（通过上下文并行CP8分布）。前向响应时通过全局softmax合并所有分片的注意力输出（`max/∑e^ℓ·o` 归约），但后向传播中KV适配器梯度未跨卡同步。
- **GLM（压缩注意力+MoE）**：78层，使用MLA（多头潜在注意力）和DSA（动态稀疏注意力），多数层为MoE。捕获后将MLA潜在页和DSA索引键页存储在**CPU内存**中，每层仅需时暂存72 MiB（IndexShare层）或88 MiB（索引计算层）到GPU。响应重放采用**完整层检查点**（reentrant checkpointing）以限制激活内存。上下文并行（CP32）与专家并行（EP32）在相同32卡上重叠使用。

**组规模扩展**：响应串行化使实时图大小仅依赖最大单个响应长度，而非组大小。因此组数从2增至8，峰值显存仅增0.208 GB（0.213%），时间线性增加。

## 6. 论文的主要结论与发现
- **可行性证明**：LongStraw在8卡H20上成功执行2.1M上下文的Qwen GRPO分组评分与反向，且近零峰值内存增长地从G=2扩展到G=8；在32卡H20上完成2.1M上下文的GLM‑5.2 78层全栈GRPO端到端执行路径。
- **瓶颈本质**：固定预算下的长上下文训练容量**受制于张量生命周期和物理所有权，而非单纯的计算稀疏性**。分离提示图的生存期，并确保物理分配与分片所有权一致，是让模型跑起来的关键。
- **架构差异分析**：密集模型（Qwen）的瓶颈在KV页存储与全局注意力的合并通信；MoE模型（GLM）尽管计算稀疏，但全序列图仍然因路由、排列和专家中间激活而爆炸，必须将提示状态驱逐到CPU并按层暂存。
- **并行维度的正交性**：上下文并行（CP）与专家并行（EP）解决不同的问题，不能相互替代，且分布式更新一致性需要单独解决。
- **系统教训**：组规模的伸缩只是调度结果，不是容量主因；层间索引共享（IndexShare）等架构特性必须反映在状态管理合同中；内存报告必须精确到物理所有权。
