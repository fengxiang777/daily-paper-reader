## 摘要
长程智能体大语言模型日益受限于有限的上下文窗口，因为延长的交互轨迹可能在任务完成前超出最大上下文长度。上下文压缩通过对先前的交互状态进行总结，并在压缩后的上下文中继续执行 rollout，提供了一种自然的解决方案，但将压缩机制融入强化学习仍处于探索不足阶段。我们提出 CompactionRL，一种通过上下文压缩训练长程智能体大语言模型的强化学习策略。我们的方法通过词元级损失归一化与跨轨迹广义优势估计，对任务执行和摘要生成进行联合优化。这一设计使得大语言模型智能体能够从压缩后的长程轨迹中学习。我们在开源模型之上训练 CompactionRL，并在智能体编码任务上观察到一致的性能提升。CompactionRL 使开源的 GLM-4.5-Air 模型（106B-A30B）在 SWE-bench Verified 上的 Pass@1 得分达到 66.8%，在 Terminal-Bench 2.0 上达到 24.5%，绝对提升分别为 7.0 和 3.1 个百分点。基于 GLM-4.7-Flash（30B-A3B），CompactionRL 将 Pass@1 提升了 5.5 和 6.8 个百分点，在 SWE-bench Verified 上达到 56.0%，在 Terminal-Bench 2.0 上达到 20.2%。因此，CompactionRL 被部署在训练开源模型 GLM-5.2（750B-A40B）的强化学习流程中。

## Abstract
Long-horizon agentic LLMs are increasingly limited by finite context windows, as extended interaction trajectories can exceed the maximum context length before a task is completed. Context compaction offers a natural solution by summarizing previous interaction states and continuing the rollout under a compressed context, but incorporating compaction into reinforcement learning remains underexplored. We propose CompactionRL, a reinforcement learning strategy to train long-horizon agentic LLMs with context compaction. Our approach jointly optimizes task execution and summary generation with token-level loss normalization and cross-trajectory generalized advantage estimation. This design enables the LLM agents to learn from compacted long-horizon trajectories. We train CompactionRL on top of open models and observe consistent performance gains on agentic coding tasks. CompactionRL enables the open GLM-4.5-Air model (106B-A30B) to achieve Pass@1 scores of 66.8% on SWE-bench Verified and 24.5% on Terminal-Bench 2.0, with absolute gains of 7.0 and 3.1 points, respectively. Built upon GLM-4.7-Flash (30B-A3B), CompactionRL improves Pass@1 by 5.5 and 6.8 points, reaching 56.0% on SWE-bench Verified and 20.2% on Terminal-Bench 2.0, respectively. CompactionRL is thus deployed in the RL pipeline for training the open GLM-5.2 model (750B-A40B).

### 1. 论文的核心问题与研究动机
- **核心问题**：长程（long-horizon）智能体大语言模型（LLM）在执行多步交互任务（如软件工程、终端操作）时，累积的推理、工具调用与观测结果很容易超出模型固定的上下文窗口，导致任务被迫中断。
- **现有方案的局限**：上下文压缩（summary/compaction）在推理时已作为启发式方法使用，但在强化学习（RL）训练中如何系统性地训练压缩策略，并使其与执行策略协同优化，仍然缺乏研究。
- **研究动机**：RL 训练中若将压缩后的轨迹段视作独立样本，会破坏时间信用分配，并引入样本权重偏差。因此，需要设计一种 RL 框架，使智能体能够在固定的上下文预算下，通过可学习的压缩策略扩展有效训练与推理范围。

### 2. 方法论：核心思想与关键技术细节
CompactionRL 基于 PPO 构建，将上下文压缩内置于 rollout 收集与优化过程中，使模型同时学习“何时以及如何总结”和“如何基于总结继续执行”。

#### 2.1 可训练的上下文压缩与轨迹分割
- **压缩触发机制**：当剩余上下文预算低于阈值 \(T_{\text{comp}}\) 时，模型在历史 \(h_t\) 上追加一个固定的总结指令 \(q_{\text{sum}}\)，并采样生成总结 \(S_t\)。
- **上下文重建**：后续推理从重建的上下文 \(\bar{h}_t\) 开始，其中包括系统提示、用户指令恢复模板（含总结）以及最近 \(k\) 步的完整交互。这一设计既保留了长程信息，又保留了局部状态。
- **轨迹划分**：一个完整的压缩轨迹被自然分割为若干个 **执行段（execution segment）** 和 **总结段（summarization segment）**，所有段的动作均由同一个策略 \(\pi_\theta\) 生成，并共享最终的轨迹级奖励 \(R(\tau)\)。

#### 2.2 针对压缩轨迹的优化策略
- **从组归一化到 PPO**：由于不同任务触发的压缩次数不同，固定规模的组归一化（如 GRPO）无法直接适配。CompactionRL 采用带 critic 的 PPO，通过价值函数为每个 segment 提供独立优势估计。
- **Token 级损失归一化**：训练时使用 token 级别的 PPO 目标函数 \(L_\pi = -\frac{1}{|M|}\sum \min(\rho \hat{A}, \text{clip}(\rho)\hat{A})\)，其中 \(M\) 为所有待优化 token 的集合。这消除了因不同 segment 数量与长度差异导致的样本权重偏差。
- **跨轨迹广义优势估计（Cross-Trajectory GAE）**：为解决 segment 独立优化带来的时间信用错配，采用校正因子对早期 segment 内的局部 GAE 优势进行折扣：
  \[
  \hat{A}_{s,i} = (\gamma\lambda)^{N_{>s}} A^{\text{loc}}_{s,i}
  \]
  其中，\(N_{>s}\) 为当前 segment 之后所有待优化 token 的总数。该校正使得优势估计与压缩前完整轨迹中该 token 距最终奖励的距离保持一致。

#### 2.3 训练流程概览
- 使用开源异步 RL 框架 slime 实现训练。
- 训练前先对 critic 进行 50 步价值预训练。
- 每轮更新采用两次价值模型更新与一次策略更新，保持 critic 与策略同步。
- 采用长度自适应 GAE (\(\lambda = 1 - 1/\alpha l\))，且 segment 内每个助手响应长度限为 10,240 token，压缩操作最多 3 次。
