## 摘要
理解并推理长上下文已成为在现实应用中部署大语言模型的关键要求。尽管最近的大语言模型支持越来越长的上下文窗口，但它们往往未能利用输入中已经存在的相关证据，揭示了上下文访问与有效上下文利用之间的差距。在本工作中，我们提出了递归证据重放作为大语言模型的长上下文推理驾驭方法（ReContext），这是一种无需训练的推理方法，用于改善长上下文推理。ReContext 使用模型内部的相关性信号构建查询条件化的证据池，并在最终生成之前重放它，同时保留完整的原始上下文。这种递归选择过程在无需训练、外部记忆或上下文修剪的情况下，将证据组织与答案生成分离。我们还提供了基于联想记忆的理论分析，将上下文描述为记忆存储，问题作为检索线索，注意力作为线索-痕迹关联，重放作为痕迹重新激活。在八个上下文长度为128K的长上下文数据集上的实验表明，ReContext 在 Qwen3-4B、Qwen3-8B 和 Llama3-8B 上一致地改善了证据利用，在三个骨干模型上取得了最佳平均排名。代码可在 https://github.com/Yanjun-Zhao/ReContext 获取。

## Abstract
Understanding and reasoning over long contexts has become a key requirement for deploying large language models (LLMs) in realistic applications. Although recent LLMs support increasingly long context windows, they often fail to use relevant evidence that is already present in the input, revealing a gap between context access and effective context utilization. In this work, we propose Recursive Evidence Replay as LLM Harness for Long-Context Reasoning (RECONTEXT), a training-free inference method for improving long-context reasoning. RECONTEXT uses model-internal relevance signals to construct a query-conditioned evidence pool and replays it before final generation while preserving the full original context. This recursive selection process separates evidence organization from answer generation without training, external memory, or context pruning. We also provide a theoretical analysis based on associative memory, which characterizes the context as a memory store, the question as a retrieval cue, attention as cue-trace association, and replay as trace reactivation. Experiments on eight long-context datasets with 128K context length show that RECONTEXT consistently improves evidence utilization across Qwen3-4B, Qwen3-8B, and Llama3-8B, achieving the best average rank on all three backbones. Code is available at https://github.com/Yanjun-Zhao/ReContext.

### 论文核心问题与整体含义
*   **核心问题**：当前的大语言模型（LLM）虽然支持越来越长的上下文窗口（如128K tokens），但存在一个显著的能力缺口：**模型能够“访问”长上下文，却无法有效“利用”其中已有的证据**。在需要密集检索和多跳推理的长文档任务中，模型常常忽略输入中已存在的相关信息，导致答案遗漏、错误或产生幻觉。
*   **整体含义**：瓶颈不仅在于上下文长度，更在于对长上下文的动态“驾驭”能力。需要一种机制，在不丢失原始信息的前提下，持续地识别、组织和更新与当前问题最相关的信息，实现更扎实、高效的长上下文推理。本工作旨在提供一种即插即用的高效增强方法，以弥合“上下文访问”与“上下文利用”之间的鸿沟。

### 方法论
论文提出一种名为 **ReContext（递归证据重放）** 的、无需训练的推理时增强方法。其核心思想是：**利用LLM内部的注意力分布作为“免费”的相关性信号，迭代筛选出与问题相关的证据，组成一个显式的“证据池”，并在最终生成答案前将该证据池重放给模型，同时完整保留原始上下文。**

关键技术流程如下：
1.  **证据筛选**：
    *   **信号提取**：给定一个长上下文 和问题 ，让LLM执行一次前向传播。选取最后几个问题侧Token（例如，最后8个）作为查询Token。
    *   **相关性评分**：计算这些查询Token在选定注意力头（Layer-Head Pairs）中对所有上下文Token的平均注意力权重，再经过指数衰减聚合，为每个上下文Token得到一个与问题相关的“相关性分数”。
    *   **Top-K选择**：仅从原始上下文Token中，挑选出相关性分数最高的 个位置，作为候选证据Token。
2.  **证据物化与重放**：
    *   **Token到语句的映射**：为避免证据碎片化，将选出的 个Token位置映射回其在原始上下文中所在的完整语句，形成一条条语义完整的证据片段。
    *   **构建证据池**：将这些证据片段按原文顺序，组成一个有序的证据池。
    *   **重放生成**：将“[原始上下文] + [证据池] + [问题]”拼接成新的提示词，输入模型，生成最终答案。此过程强调“强调而非排除”，原始上下文完整保留。
3.  **递归迭代（关键创新）**：
    *   **递归筛选**：上述过程并非一次性完成。在第一轮筛选出证据池 后，第二轮会将“[原始上下文] + [证据池 ] + [问题]”作为新的输入执行前向传播。此时，重放的证据会改变模型内部状态，使得下一轮的注意力评分能发现与已有证据相关的新证据。
    *   **证据更新**：在后一轮中，仅保留新选择的、且未出现在当前证据池中的语句，并追加到证据池末尾，实现证据池的有序扩充。此过程可重复 轮。
    *   **理论支撑**：论文从“联想记忆”视角提供了一个理论分析（单调性改进定理），将上下文视为记忆存储，问题作为检索线索，注意力作为线索-记忆痕迹的关联，证据重放作为痕迹再激活。理论上证明，这种递归重放能单调地推动模型的隐藏表征向正确答案的嵌入向量靠拢。

### 优点
*   **无需训练，即插即用**：ReContext是一种推理时增强方法，不改变模型参数，可直接应用于现有LLM，使用简便。
*   **无损证据增强**：它保留了完整的原始上下文，将筛选出的证据作为“强调”而非“替代”，避免了信息丢失风险，比上下文压缩或检索方法更安全。
*   **显式证据组织**：将模型内部的隐式相关性信号转化为显式、可读的证据池，增强了推理过程的可解释性和可控性。
*   **理论支撑**：从联想记忆和查询-证据再绑定的角度提供了理论解释，并给出了单调性改进证明，为方法有效性提供了理论基础。
*   **高效的证据复用**：递归过程中可复用原始上下文的KV缓存，仅需对新增证据和问题部分进行计算，实现了轻量级的迭代优化。

### 不足与局限
*   **对模型内部信号的依赖**：方法需要访问模型内部的注意力权重，因此无法直接应用于仅提供API接口而不暴露内部信号的闭源商业模型（如GPT-4）。
*   **推理延迟增加**：方法增加了额外的证据筛选与重放前向过程，因此推理耗时高于直接解码的Vanilla基线（尽管仍快于某些修改解码逻辑的方法如DySCO）。
*   **证据选择精度风险**：实验显示，候选Token预算是一个任务敏感的超参数。过大的 可能引入噪声，对某些任务（如NQ）反而造成性能下降，存在精确度与召回率之间的权衡。
*   **超参数依赖**：递归轮数 、候选Token预算 等核心超参数的最优设定是任务依赖的，在实际应用中可能需要少量的手动调整。
