## 摘要
大型语言模型（LLMs）展示了强大的推理和生成能力，但其固定的上下文窗口限制了跨会话交互的长期信息积累和重用。现有的记忆增强系统通常以粗糙和不稳定的方式构建记忆，依赖于低效的记忆表示或不稳定的无约束更新。为了解决这些挑战，我们提出了AtomMem，一个专为价值密集存储和稳定记忆演化而设计的长期记忆系统。AtomMem引入了一个事实执行器，它从长形式交互中选择性地提取高价值的原子事实，作为高效记忆表示。随后，AtomMem将这些事实组织成层次化的事件结构和时间画像，捕获连贯的情节上下文，并跟踪随时间动态演变的用户属性。在检索时，系统激活一个联想记忆图来连接碎片化的记忆。在LoCoMo基准上的实验证实，AtomMem在各种推理任务中实现了最先进的性能，为部署智能个性化代理提供了一种可扩展且经济可行的解决方案。

## Abstract
Large language models (LLMs) demonstrate strong reasoning and generation abilities, but their fixed context windows limit long-term information accumulation and reuse across multi-session interactions. Existing memory-augmented systems often construct memory in a coarse and unstable manner, relying on inefficient memory representations or unstable unconstrained updates. To address these challenges, we propose AtomMem, a long-term memory system designed for value-dense storage and stable memory evolution. AtomMem introduces a Fact Executor, which selectively extracts high value atomic facts from long form interactions to serve as highly efficient memory representations. Subsequently, AtomMem organizes these facts into hierarchical event structures and temporal profiles, capturing coherent episodic contexts and tracking dynamically evolving user attributes over time. During retrieval, the system activates an associative memory graph to connect fragmented memories. Experiments on the LoCoMo benchmark confirm that AtomMem achieves state-of-the-art performance across various reasoning tasks, offering a scalable and economically viable solution for deploying intelligent personalized agents.

## 1. 核心问题与整体含义
*   **核心痛点**：大型语言模型（LLM）的固定上下文窗口限制了其在多轮、跨会话交互中积累与复用长期信息的能力，导致遗忘用户偏好、重复已解决问题、事实矛盾等问题。
*   **现有记忆系统的局限**：目前记忆增强系统多以粗糙（如直接存储原始对话）或不稳定（无约束的LLM重写）的方式构建记忆，面临“高信息密度 vs. 上下文保真度”的两难：原始对话有噪声冗余，压缩表示易丢失细节并引入LLM自身噪声，且记忆更新缺乏稳定性。
*   **整体含义**：提出 **AtomMem**——一个以“原子事实”为核心记忆表征的长期记忆框架。它将无结构对话流转化为紧凑、自包含且可检索的结构化单元，并组织为层次化事件结构与动态时间画像，再通过联想图检索实现高效、稳定的记忆管理，使LLM智能体在长程交互中保持连贯、一致的个性化理解。

## 2. 方法论
AtomMem的整体流程包括：基本表示（原子事实提取）、情节整合（事件记忆构建）、状态演化（时间画像建模）、联想召回（记忆图激活）以及层次化检索与响应生成。

*   **原子事实提取（基本表示）**
    *   **Fact Executor**：基于监督微调（SFT）的轻量LLM（Qwen3-14B），将带噪原始对话转换为独立、自包含的原子事实。该模块完成去噪、指代消解、时间锚定等轻量推理。
    *   **结构化事实构造**：提取的事实被封装为`F = {id, c, v, P, K, T, E}`，其中`c`为文本内容，`v`为稠密语义向量，`P`为参与者，`K`为关键词，`T`为时间信息，`E`链接至事件。
    *   **事实验证**：在存储前，通过参与者、时间等符号过滤得到候选集，使用混合相似度 `S_h(x,y) = α·sim_e(v_x,v_y) + β·Jac(K_x,K_y)` 检索Top-k相关事实；再由LLM判断新事实与已有事实的关系，仅存储非冗余的残留内容，并在检测到冲突时生成更新指令。

*   **事件记忆构建（情节整合）**
    *   将相关原子事实聚合并为“事件块” `E = {id, S, F_ids, P_e, K_e, T_e}`。
    *   对于每个新事实，基于已检索的相关事实获取候选事件或独立事实，LLM判断其逻辑归属，决定吸收到现有事件、触发创建新事件或独立存储，并同步更新事件摘要、关键词、参与者、时间跨度。

*   **时间画像建模（状态演化）**
    *   刻画用户稳定但可变的属性（如偏好、习惯）：`P = {id, u, c, v_p, K_p, E_evi, t_from, H}`，`H`保存历史版本。
    *   采用**会话级批量更新**：LLM识别可能反映长期特性的事实并加入等待队列，会话结束时批量处理，通过混合相似度检索最相关现有画像，LLM驱动的更新器决定是否新增、更新或标记为冗余，并保留历史版本时间线。

*   **联想记忆图激活**
    *   以原子事实为节点，构建三类边：（1）**实体边**：基于IDF加权关键词重叠，权重`w_kw`使用查询感知的`ω(k)`增强；（2）**事件边**：两事实同属一个事件，权重按事件大小衰减；（3）**时间边**：同一会话中相邻对话轮次的事实，权重按轮次距离指数衰减。
    *   检索时，以初召回的种子事实构建局部子图，通过**个性化PageRank（带重启的随机游走）** 传播激活值，重启分布由种子得分经幂变换得到，最终按稳态激活分排名，选出Top-k事实。

*   **层次化检索与响应生成**
    *   **查询意图解析**：LLM解析查询得到结构化对象`Q_parsed`（是否需要画像、参与者、关键词、时间范围）。
    *   **三阶段召回**：
        1.  **主召回**：用元数据快速过滤，通过`S_h(F, Q)`排名得到初选集`R_pri`。
        2.  **补偿召回**：召回事件层，用`S_h(E, Q)`找到Top事件，提取其中事实，通过融合分数`S_f = w_e·S_h(E,Q) + (1-w_e)·S_h(F,Q)`排名，形成`R_comp`，与`R_pri`合并为`R_seed`。
        3.  **联想召回**：以`R_seed`为种子激活记忆图，执行RWR，取高激活分事实得最终事实集`R_fact`。
    *   **画像增强与生成**：若需要画像，用`S_h(P, Q)`检索画像，并根据时间约束选择合适的版本。将`R_fact`与画像上下文`R_prof`拼接，交由LLM生成最终回答。

## 6. 主要结论与发现
*   AtomMem在所有LoCoMo任务（单跳、多跳、时间推理、开放域）上取得**最优（SOTA）** 或次优性能，尤其在需要长上下文整合的复杂推理任务上优势显著（多跳J-score比强基线LightMem高5.5%，时间推理高31.1%）。
*   **原子事实表征质量是核心**：简化的AtomMem-Flat（不含层次结构）仅靠检索原子事实即可大幅超越使用原始对话的LoCoMo基线（多跳F1提升76.6%），且Token消耗极低（722k vs. 827k），证明结构化、紧凑的事实单元是高效记忆的基础。

## 7. 优点
*   **表征创新**：提出“原子事实”作为最小语义单元，从根本上解决了记忆存储的冗余与不可靠问题，实现了高信息密度的稳定表征。
*   **架构全面**：将原子事实组织为事件（情节整合）和动态时间画像（状态演变），并引入多通道联想图（实体/事件/时间）进行类人记忆召回，有机融合了细粒度细节与高层上下文。
*   **高效稳定**：通过SFT的Fact Executor和LLM驱动的验证与更新机制，使记忆存储和更新更加可控、稳定，避免了无约束LLM重写的漂移问题。
*   **实验扎实**：在权威长程基准上进行了公平、多维的对比和消融，且SFT数据隔离设计确保了结果可信度。同时公开了代码和高质量提取数据集。

## 8. 不足与局限
*   **LLM依赖性**：多阶段（事实提取、验证、事件构造、意图解析、判定与生成）都依赖基础LLM的能力，其性能受底层模型生成稳定性的限制。
*   **模态单一**：当前系统仅处理文本交互，无法直接处理多模态输入（如图像、音频），限制了在真实世界多模态对话中的应用。
*   **计算效率有优化空间**：虽然整体Token效率优于同级别系统，但文中承认Token效率仍可进一步优化；同时记忆构造带来的部分开销未被完全消除。
*   **未深入讨论隐私/安全**：对于高度个人化的记忆（如用户偏好、经历）在存储与处理中的隐私保护和信息安全性未作讨论。
*   **评估的泛化性**：虽有两个数据集，但LongMemEval结果主要见于附录，且未在更多样化的个性化或动态环境（如角色扮演）中检验鲁棒性。
