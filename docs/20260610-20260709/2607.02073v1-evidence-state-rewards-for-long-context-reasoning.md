## 摘要
长上下文推理要求模型定位、修正并综合分布在冗长输入中的证据。现有的长上下文强化学习方法通常只奖励最终答案或静态证据抽取，对中间动作如何改变模型的证据状态提供的反馈很少。我们提出了 Maven，一个具有可编辑证据记忆的强化学习框架。Maven 定义了一个以答案为条件的证据状态价值，并对动作级的状态转移进行奖励：添加动作根据边际收益和事后贡献得到奖励，链接动作根据证据协同效应得到奖励，丢弃动作根据移除误导证据后答案支持的改善得到奖励。这些奖励被分配给 GRPO 中相应的动作片段。在 LongBench v2、LongReason 和 RULER 上对 Llama 和 Qwen 模型的实验表明，Maven 优于仅奖励结果的强化学习和证据识别基线，能产生更充分的证据集和更低的干扰项保留。我们的结果表明，长上下文强化学习受益于优化有状态的证据导航，而非一次性证据抽取。

## Abstract
Long-context reasoning requires models to locate, revise, and synthesize evidence distributed across lengthy inputs. Existing long-context RL methods usually reward final answers or static evidence extraction, offering little feedback on how intermediate actions change the model's evidence state. We propose Maven, a reinforcement learning framework with an editable evidence memory. Maven defines an answer-conditioned evidence-state value and rewards action-level state transitions: add actions are credited by marginal gain and hindsight contribution, link actions by evidence synergy, and drop actions by improved answer support after removing misleading evidence. These rewards are assigned to the corresponding action spans in GRPO. Across Llama and Qwen models on LongBench v2, LongReason, and RULER, Maven outperforms outcome-only RL and evidence-identification baselines, producing more sufficient evidence sets and lower distractor retention. Our results show that long-context RL benefits from optimizing stateful evidence navigation rather than one-shot evidence extraction.

### 1. 核心问题与整体含义

- **研究背景与动机**：尽管大语言模型（LLM）的上下文窗口不断增长，但在长上下文中进行有效推理仍然困难。模型容易出现“迷失在中间”、过早依赖部分证据、无法综合分散信息等问题。
- **现有方法的不足**：当前用于长上下文推理的强化学习方法主要关注最终答案的奖励，或对证据抽取进行静态监督（如要求模型识别特定段落）。这类方法忽略了推理过程中证据状态的动态变化：证据往往需要组合才产生价值，误导性信息需要在获取新证据后被丢弃。
- **整体含义**：本文旨在将长上下文推理建模为一种有状态的证据记忆导航过程，通过奖励每一步动作对证据状态的影响，而非仅仅奖励最终答案或一次性证据抽取的质量。

### 2. 方法论

Maven 的核心思想是：让策略模型通过一系列结构化动作（添加、链接、丢弃、回答）来编辑一个证据记忆，并基于证据记忆对正确答案预测能力的改善为每个动作分配奖励。

- **可编辑证据记忆与动作类型**：
  - `<add id=i>`：将上下文中指定 id 的证据片段加入记忆，并简述其相关性。
  - `<drop id=i>`：从记忆中移除无用、冗余或误导的证据。
  - `<link ids=i,j>`：解释两个证据如何协同支持答案。
  - `<answer>`：在收集足够证据后给出最终答案。
  模型可自由组合动作顺序，最终形成的证据集记为 \(E_F\)。

- **答案条件化的证据状态价值**：使用一个冻结的验证器模型 \(p_\psi\)，计算在给定问题 \(Q\) 和当前证据记忆 \(E\) 的条件下，生成正确答案 \(a^\star\) 的负对数似然（NLL）减少比例，作为证据状态价值：
  \[
  V_\psi(Q, E) = \frac{\ell_\psi(Q,\emptyset) - \ell_\psi(Q, E)}{\ell_\psi(Q,\emptyset) + \epsilon}
  \]
  数值越大代表证据对答案的预测支持越强。

- **动作奖励设计**：
  - **添加奖励**：由两部分加权组合（平衡系数 \(\alpha\)）。
    1. 在线条件信息增益 (CAIG)：\[CAIG_k(e_k) = V_\psi(Q, E_k) - V_\psi(Q, E_{k-1})\] （衡量新证据对当前状态的直接提升）。
    2. 事后贡献 (HCC)：若证据 \(e_i\) 最终被保留，计算移除它后最终状态价值的下降量，作为其对最终证据集的边际贡献。
  - **丢弃奖励**：\[V_\psi(Q, E_k) - V_\psi(Q, E_{k-1})\] 若丢弃后状态价值上升，则奖励为正（移除干扰有效），否则为负。
  - **链接奖励**：对两个被保留的证据 \(e_i, e_j\)，计算协同分数：
    \[
    \text{Syn}(e_i,e_j; E_F) = V_\psi(E_F) - V_\psi(E_F\setminus\{e_i\}) - V_\psi(E_F\setminus\{e_j\}) + V_\psi(E_F\setminus\{e_i,e_j\})
    \]
    若协同为正，则给予正向奖励，鼓励模型发现互补证据。
  - **回答奖励**：基于最终答案与标准答案的子串匹配，提供二元奖励。

- **训练过程**：
  - 采用 GRPO 优化策略，将上述动作级奖励按动作类型进行组内归一化，并分配到对应的生成 token 片段上作为优势信号。
  - 训练分两阶段：先进行少量 SFT 冷启动，教模型使用动作接口；再进行 RL 训练，早期使用较短上下文和较简单问题稳定探索。

### 3. 实验设计

- **训练数据集**：
  - 规模：9K 条长上下文问答对，上下文长度 8K–64K tokens。
  - 来源：3K 来自 LongRLVR（经筛选确保多证据依赖，去除单证据捷径），6K 从 HotpotQA、2WikiMultiHopQA、MuSiQue 构造（加入随机干扰项和老师模型生成的硬干扰项）。
  - SFT 冷启动数据：2K 条同样来源的示例，其中 ≤20% 包含故意添加后再丢弃的干扰项动作。

- **基准模型与对比方法**：
  - 基模型：Llama-3.1-8B，Qwen2.5-14B，Qwen3-30B-A3B。
  - 消融对比：原始基模型、冷启动 SFT 模型、仅结果奖励 RL、结果 + 证据识别 F1 奖励 RL、结果 + 证据识别 + 探索提示 RL。
  - 参考模型：Llama-3.1-70B，Qwen3-32B（Thinking），QwenLong-L1-32B。

- **评测基准**：
  - LongBench v2（短、中、长上下文子集），LongReason（32K、64K、128K），RULER（部分子集，64K 和 128K）。
  - 诊断评估集：150 个构造样本，评估证据充分率（最终保留证据覆盖金标准证据的比例）和干扰项保留率。

### 4. 资源与算力

- **GPU 型号与数量**：使用 8 块 NVIDIA A100 和 4 块 AMD MI250X GPU。
- **训练耗时**：三个模型（8B、14B、30B）的 SFT 和 RL 训练总计约 1,250 GPU 小时（不同硬件环境取均值）。
- **推理效率**：训练时冻结的验证器仅在紧凑的证据片段和问题上计算答案 NLL，不参与长文本生成，且限制动作次数以减少调用量；推理时验证器不使用，因此无额外开销。

### 5. 实验数量与充分性

- **实验组合丰富**：
  - 在三个不同规模、不同家族的基模型上重复主要实验。
  - 对每个基模型进行了包括冷启动 SFT、三种 RL 基线在内的 6–7 种配置比较。
  - 设有训练动态监控（每隔 50 步在固定子集上评测 LongBench v2 和诊断指标）。
  - 消融实验：移除添加奖励、移除链接奖励、移除丢弃奖励、仅基于最终证据集奖励的变体；此外还分析了验证器选择、对比式答案打分、奖励权重超参数（含动作权重比例、α 值）的影响。
  - 在短上下文推理任务（MMLU-Pro）上检查训练是否损害通用能力。

- **公平性与客观性**：
  - 所有对比方法使用相同的训练数据、基模型和优化器设置。
  - 使用外部基准的官方评测配置，结果取三次运行平均。
  - 诊断指标的评估基于统一要求输出证据标识符的方式，保证测量一致性。

总体来说，实验覆盖充分，多模型、多基准、多角度的比较和消融验证了方法的有效性和健壮性。

### 6. 主要结论与发现

- Maven 在所有三个基模型、所有三个长上下文基准上均一致优于各 RL 基线（尤其在长上下文子集上提升明显）。
- 相较于结果 + 证据识别奖励，Maven 不仅提高了证据充分率（找到更多所需证据），更显著降低了干扰项保留率，表明模型学会了修正和丢弃误导信息。
- 动作奖励缺一不可：添加奖励缺失导致最大性能下降，丢弃奖励缺失时干扰项保留最高，链接奖励缺失影响证据充分度；仅基于最终证据集的奖励效果远弱于动作级过程奖励。
- 训练动态显示 Maven 提升更稳定且上限更高，而证据识别奖励虽提升证据充分性却未能有效降低干扰项。
- 对短上下文通用推理能力无明显损害。

### 7. 优点

- **新颖的过程奖励设计**：首次将长上下文推理视为有状态证据记忆编辑，用答案条件化的价值函数为添加、链接、丢弃动作提供局部奖励，克服了以往静态证据抽取奖励的局限。
- **模块化且可解释的动作结构**：模型输出的结构化动作兼具推理链作用，便于分析与监督。
- **无需训练额外过程奖励模型**：仅用一个冻结的轻量级验证器计算 NLL，训练成本可控。
- **多模型、多基准的全面验证**：在 8B/14B/30B 模型和多个主流长上下文评测集上取得稳健提升，并提供了丰富的诊断与消融分析。
- **兼顾证据充分性与干扰项清除**：有效缓解了模型过早锚定误导性证据的问题。

### 8. 不足与局限

- **依赖标准答案**：训练时需金标准答案来计算证据状态价值，难以直接扩展到开放式生成或无明确答案的对话、创作任务。
- **证据记忆接口假设**：要求证据能表示为有边界的文本片段，对于图像、图表等多模态证据或长散文式证据支持尚不适用。
- **动作语义的局限性**：链接和丢弃的定义依赖最终保留的证据集，对中间推理步骤中动态形成、后又抛弃的假设联结无法奖励。
- **评估粒度**：诊断分析基于自动构造的干扰项和标准答案，缺乏人工对证据质量、忠实度和正确性的全面评估。
- **计算效率**：虽然验证器相对轻量，但每条轨迹仍需多次调用，当证据项较多时开销有所上升；动作预算限制了证据记忆大小。
- **训练数据分布**：训练数据包含多跳 QA 构造数据，可能与某些真实应用场景存在分布差异，泛化性需进一步验证。
