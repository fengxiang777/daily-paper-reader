## 摘要
用于大型语言模型（LLM）智能体的长期记忆系统通常试图记住一切，无差别地提取记忆以尽可能多地保留事实。然而在实际应用中，推理成本和有限的上下文预算使这种做法不可行：除了将原始对话整合到记忆中之外，智能体必须实施写入控制，高效地只保留每个用户真正关心的信息。否则，长程个性化交互将遭遇记忆膨胀，无关琐事会挤占有用信息，逐渐侵蚀问答准确性。我们认为，值得记住的内容是角色相关的，并提出AdaMem（自适应记忆），一种从反馈中为每个用户学习该记住什么的方法。AdaMem维护一个结构化的、角色特定的记忆策略，并通过轻量级的补丁式自我反思步骤和失败回滚机制，利用每周的问答反馈来改进该策略。为了研究这一设定，我们构建了AdaMem-Bench，一个模拟多周交互及每周问答的基准测试。在两个提取模型和两种反馈模式下，与均匀提取的Mem0基线相比，AdaMem使问答准确率最高提升9.0%，同时将记忆体积缩减9%。

## Abstract
Long-term memory systems for Large Language Model (LLM) agents typically try to \emph{remember everything}, extracting memories uniformly to retain as many facts as possible. In production, however, inference cost and finite context budgets make this untenable: beyond consolidating raw dialogue into memory, an agent must exert \emph{write control}, efficiently keeping only the information each user actually cares about. Otherwise, long-horizon personalized interactions suffer \emph{memory bloat}, where irrelevant trivia crowds out useful information and steadily erodes question-answering (QA) accuracy. We argue that what is worth remembering is role-dependent, and propose \textbf{AdaMem} (Adaptive Memory), a method that \emph{learns what to remember} for each user from feedback. AdaMem maintains a structured, role-specific Memory Policy and refines it from weekly QA feedback through a lightweight, patch-style self-reflection step with failure rollback. To study this setting, we build \textbf{AdaMem-Bench}, a benchmark that simulates weeks of interaction with week-by-week QA. Across two extraction models and two feedback modes, AdaMem improves QA accuracy by up to \textbf{+9.0\%} over the uniform Mem0 baseline while shrinking memory volume by \textbf{9\%}.

### 1. 论文的核心问题与整体含义（研究动机和背景）
这篇论文针对的是基于大型语言模型（LLM）的个性化智能体在长期交互中的 **记忆膨胀（Memory Bloat）** 问题。当前主流的长期记忆系统（如Mem0）倾向于**记住一切**，无差别地提取和存储所有对话信息。这种做法在实际部署中面临严峻挑战：

-   **成本与容量限制**：推理成本和上下文窗口有限，不可能无限制地存储所有记忆。
-   **性能衰减**：随着时间推移，无关琐事（如日常寒暄、不重要的日程）会充满记忆库，挤占和干扰真正重要的信息，导致智能体在回答个性化问题时准确率持续下降。

论文的核心主张是：在长程交互中，**值得记忆的内容是角色相关的（Role-dependent）**。一个理想的智能体应当学习每个用户真正关心什么，并实施 **写入控制（Write Control）** ，只保留用户偏好的信息。例如，与经理交谈应关注商业决策，与健身伙伴交谈则应关注健身计划。因此，研究的目标是让智能体学会“**记住什么**”，以维持高效、精准且低成本的个性化服务。

### 2. 论文提出的方法论：AdaMem
论文提出了 **AdaMem（自适应记忆）** 方法，核心思想是通过反馈来动态学习并维护一个**角色特定的记忆策略（Memory Policy）**，用于指导记忆的提取过程，从而实现从“什么都要记”到“只记有用的”转变。

**核心流程**
AdaMem 采用一个闭环的**每周循环**，包含四个关键阶段：

-   **每日提取（Daily Extraction）**：根据当前激活的角色记忆策略，从当日对话中提取符合策略的记忆条目，并存入记忆库。此过程包含**时间锚定（Time Anchoring）**，即强制将“明天”这类相对时间表达解析为绝对日期（如“2026年6月18日”）后再存储。
-   **每周评估（Weekly Evaluation）**：在周末，通过一组问答问题来探测智能体的记忆效果。
-   **策略反思（Policy Reflection）**：对答错的问题进行诊断，分析失败原因，并生成一个“补丁”来更新记忆策略。
    -   **故障诊断**：将错误分为两类——**召回失败**（策略过于严苛，漏掉了应记的信息）和**精确度失败**（策略过于宽松，存入了噪声记忆并误导了回答）。
    -   **补丁式更新**：根据诊断结果，生成自然语言形式的策略调整建议（∆Pt）。
    -   **失败回滚**：为了确保稳定性，新策略会在历史问答对上进行验证。如果效果变差，则回滚到旧策略，防止策略偏移。
-   **策略渲染（Policy Rendering）**：将更新后的记忆策略编译为下一次每日提取时使用的具体指令。

**记忆策略结构**
记忆策略是一个映射关系，将每个角色映射到其对应的自然语言偏好描述。例如：“`角色_经理`：只提取最终商业决策和截止日期；忽略日常闲聊。”
