## 摘要
语言模型代理通过多轮工具使用在解决现实任务方面日益有效。然而，训练可靠的工具使用代理在实践中仍具挑战性。虽然强化学习提供了一种从自身环境交互中改进代理的同策略范式，但其有效性高度依赖于训练任务分布。当任务在训练前固定时，任务分布可能与策略不断发展的能力越来越不匹配，导致许多推演过程被花费在无信息的任务上。我们提出SENTINEL，一个将求解器的推演失败转化为针对性训练任务的失效驱动强化学习框架。SENTINEL遵循控制器-提议器-求解器循环：控制器分析失败的轨迹并归纳重复出现的错误模式，提议器生成突出这些弱点的可执行任务，求解器在有针对性的任务上进行训练。在Tau2-Bench零售场景中，使用Qwen3-4B-Thinking-2507模型，SENTINEL将Pass^1从66.4提高到74.9，并且在Pass^k指标上优于基于一般合成任务的强化学习。这些结果表明，模型失效为改进工具使用语言模型代理提供了有效且可扩展的定向训练信号来源。

## Abstract
Language model agents are increasingly effective in solving realistic tasks through multi-turn tool use. However, training reliable tool-using agents remains challenging in practice. While reinforcement learning provides an on-policy paradigm for improving agents from their own environment interactions, its effectiveness depends heavily on the training task distribution. When tasks are fixed before training, the task distribution can become increasingly mismatched with the policy's evolving capabilities, causing many rollouts to be spent on uninformative tasks. We propose SENTINEL, a failure-driven reinforcement learning framework that turns the Solver's rollout failures into targeted training tasks. SENTINEL follows a Controller--Proposer--Solver loop: the Controller analyzes failed trajectories and summarizes recurring error patterns, the Proposer generates executable tasks that stress these weaknesses, and the Solver is trained on the targeted tasks. On Tau2-Bench Retail with Qwen3-4B-Thinking-2507, SENTINEL improves Pass\^{}1 from 66.4 to 74.9 and outperforms RL on general synthetic tasks across Pass\^{}k metrics. These results demonstrate that model failures provide an effective and scalable source of targeted training signal for improving tool-using language model agents.

*   **核心痛点**：传统的RL方法依赖一个在训练开始前就固定好的任务集。随着模型策略在学习过程中不断进化，这个固定的任务分布可能变得与模型当前的实际弱点脱节。这会导致大量RL训练推演（rollouts）被浪费在过于简单、过于困难或重复的信息量低的任务上，甚至鼓励模型学习利用奖励信号的捷径，而非真正掌握健壮的工具使用行为。

### 2. 论文提出的方法论

论文提出了名为SENTINEL的**失效驱动强化学习**框架，其核心是一个“控制器-提议器-求解器”的闭路迭代循环。

*   **核心思想**：利用代理（求解器）自身的失败推演作为反馈，来动态生成下一轮RL训练所需的、针对其当前弱点的任务。
*   **关键技术流程**：
    1.  **求解器推演**：当前策略在可执行工具环境中执行任务，生成轨迹。
    2.  **控制器分析**：
        *   **轨迹筛选**：控制器定期触发，收集失败的轨迹和部分成功轨迹。
        *   **失效模式发现**：利用大模型分析失败的轨迹，对比其与黄金操作，识别出重复出现的错误模式。
        *   **生成指令**：结合当前的错误模式、已掌握的技能和跨轮次的训练历史，生成一份自然语言的“指令”，指导下一阶段的任务生成。指令会优先处理持续存在的顽固弱点，并降低已解决模式的优先级。
    3.  **提议器生成**：
        *   **轨迹接地构建**：为避免生成不可执行的任务，提议器首先在环境中探索并构建一条可行的工具调用轨迹作为“参考轨迹”。
        *   **失败感知重写**：然后，提议器在控制器指令的引导下，将该参考轨迹重写为一个面向用户的自然语言任务。重写过程会刻意突出控制器识别的弱点。
    4.  **求解器训练**：生成的目标性任务被注入训练数据缓冲区。求解器使用GRPO算法在由原始任务和新生成任务组成的缓冲区上进行强化学习优化。奖励函数除了最终任务成功信号，还包含执行检查、重复调用惩罚、格式惩罚等塑形项，以减少奖励黑客行为。
*   **循环迭代**：训练后的求解器再次进行推演，其新的失败轨迹被送回控制器，开始新一轮的闭环训练。

### 6. 论文的主要结论与发现

*   **性能提升**：SENTINEL在Tau2-Bench零售领域显著提升了工具使用代理的性能。例如，Pass^1从基线模型的66.4提升至74.9，并全面超越在通用合成任务上训练的强化学习基线。
*   **失效驱动有效**：通过将训练任务分布动态调整为针对当前策略的失败模式，能比使用静态的、通用的任务分布更有效地进行强化学习。
*   **对强基线的增益**：即使对于经过监督微调、性能已经很强的模型，失效驱动的RL仍能带来进一步改进，而直接在此基础上应用通用RL则可能导致性能下降。这说明，当模型具备一定能力后，针对其剩余弱点的定向训练至关重要。
*   **失败模式洞察**：代理的失败主要集中在更精细的层面，如准确选择物品变体、完成必需中间步骤、遵循领域策略等，而非简单的工具选择错误。SENTINEL能够适应这种模式变化，随训练进行逐步聚焦于更高阶的策略性错误。

### 7. 优点

*   **创新的闭环设计**：提出了一种从失败中自我演进的闭环训练框架，将失败诊断、任务生成与策略优化无缝连接，使训练过程更具适应性和可控性。
*   **任务生成的接地性**：采用“先构建可行轨迹，再重写为任务”的方式，确保了生成任务的可执行性和自洽性，解决了直接生成任务可能存在的不可达或指令-答案不匹配问题。
*   **针对性强**：明确将训练任务与模型的具体弱点绑定，实现了高效的定向学习，避免了在已掌握技能上的无效训练，直观且符合认知逻辑。
*   **实验设计严谨**：充分考虑了基线对比的公平性，并通过在SFT强模型上的实验验证了方法的普适增益，超越了简单的低基线提升。

### 8. 不足与局限

*   **领域验证有限**：当前实验仅在Tau2-Bench零售这一个可控领域进行。该框架能否有效推广到具有不同动态、工具集和任务结构的其他领域尚待验证。
*   **依赖基模型能力**：失效模式的分析和任务的生成都高度依赖于作为控制器和提议器的LLM的能力。基模型的能力上限可能成为方法性能的瓶颈，尤其在针对其无法诊断的细微错误时。
*   **强化学习的固有挑战**：在强基线上进行RL存在训练不稳定导致性能崩溃的风险（论文中SFT+RL对照组的性能下降也佐证了此点）。SENTINEL虽然缓解了此问题，但在实际部署中仍可能面临优化不稳定、奖励塑形复杂等挑战。
*   **缺乏组件消融**：未通过实验量化控制器、提议器、各类奖励塑形项等各个组件的独立贡献，对框架内部工作机制的解释不够完整。
*   **覆盖局限性**：生成的针对性任务基于已暴露的失败，可能无法覆盖所有潜在的风险测试案例，尤其是一些罕见或未被探索到的场景。
